## 1. Зачем

### Текущие проблемы
- **16 500 строк** vanilla JS в 67 файлах-модулях, конкатенируемых в один `<script>`
- Нет компонентной архитектуры — всё в глобальном scope, функции зависят друг от друга неявно
- Нет реактивности — обновление UI вручную через `innerHTML` и `document.getElementById`
- Шаблоны внутри JS-строк (`'<div>' + ... + '</div>'`) — хрупко, нечитаемо, баги с экранированием (`\'`, `\\n`)
- Нельзя использовать готовые библиотеки компонентов (таблицы, рисовалки, rich-text редакторы)
- Невозможен hot-reload при разработке — каждое изменение = перезапуск сервера

### Что даст миграция
- **Компоненты**: переиспользуемые, тестируемые, изолированные
- **Экосистема**: Excalidraw, TipTap (rich-text), TanStack Table, React-DnD, и т.д.
- **DX**: hot-reload, TypeScript, автокомплит, нормальный debugging
- **Качество**: JSX вместо строковых шаблонов, типизация, линтинг JSX
- **Скорость разработки**: новые фичи в 2-3 раза быстрее

## 2. React vs Vue

| Критерий | React | Vue |
|----------|-------|-----|
| Экосистема | Огромная, больше всего библиотек | Большая, но меньше чем React |
| Кривая обучения | Средняя | Ниже (ближе к HTML) |
| Excalidraw | ✅ Нативно (React) | ❌ Нужна обёртка |
| TipTap (rich-text) | ✅ | ✅ |
| Сообщество | Крупнейшее | Второе по размеру |
| TypeScript | Отлично | Отлично |
| AI-генерация кода | Лучше (больше примеров) | Хорошо |

**Рекомендация: React** — из-за Excalidraw (для заметок) и самой большой экосистемы.

## 3. Стек

| Слой | Инструмент | Зачем |
|------|-----------|-------|
| Фреймворк | **React 19** | Компоненты, реактивность |
| Язык | **TypeScript** | Типизация, меньше багов |
| Сборка | **Vite** | Мгновенный HMR, быстрая сборка |
| Роутинг | **React Router v7** | SPA-навигация (замена нашего hash-роутинга) |
| Стейт | **Zustand** | Простой глобальный стейт (замена глобальных переменных) |
| Запросы | **TanStack Query** | Кеширование, загрузка, ревалидация API |
| Стили | **Tailwind CSS** или **CSS Modules** | Консистентность, нет конфликтов |
| Таблицы | **TanStack Table** | Сортировка, фильтрация, пагинация |
| Rich-text | **TipTap** | Для заметок — текстовые блоки |
| Рисование | **Excalidraw** или **tldraw** | Для заметок — canvas |
| Иконки | **Lucide React** | Уже используем Lucide |
| Формы | **React Hook Form + Zod** | Валидация форм |
| Тесты | **Vitest + React Testing Library** | Замена Jest |

## 5. Стратегия миграции: постепенная (Strangler Fig)

**НЕ переписываем всё сразу.** Мигрируем по одной странице, старый и новый код живут рядом.

### Фаза 0: Подготовка (1 неделя)
- [ ] Создать `frontend/` с Vite + React + TypeScript
- [ ] Настроить `vite.config.ts` с proxy на `http://localhost:3002/api`
- [ ] Перенести CSS-переменные и базовые стили
- [ ] Создать `api/client.ts` (обёртка fetch с auth + refresh token)
- [ ] Создать `stores/authStore.ts` (логин, токены, TOTP)
- [ ] Создать Layout: Sidebar + Topbar (как есть)
- [ ] Настроить Nginx: `/app` → Vite (3000), `/api` → Express (3002), `/` → старый HTML
- [ ] **Результат**: две версии живут параллельно, можно тестировать

### Фаза 1: Простые страницы (2 недели)
- [ ] **Логин** — форма + TOTP
- [ ] **Заметки** — Excalidraw/TipTap (самая выигрышная миграция)
- [ ] **Настройки** — типы и поля
- [ ] **Карта** — статический img + SVG overlay
- [ ] **Результат**: 4 страницы на React, остальные на старом коде

### Фаза 2: Списки и карточки (3 недели)

## 7. Риски

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Регрессии при миграции | Высокая | Постепенная миграция, старый код работает параллельно |
| Увеличение размера бандла | Средняя | Code splitting, lazy loading, tree shaking |
| Сложность деплоя | Низкая | Vite build → статика, nginx раздаёт |
| Потеря мгновенного деплоя | Средняя | `vite build` ~5 сек, автоматизировать в systemd |
| Новые зависимости = новые уязвимости | Средняя | npm audit, Dependabot, lockfile |

## 8. Деплой (после миграции)

```
# Текущий деплой (мгновенный):
systemctl restart indparkdocs

# Новый деплой:
cd frontend && npm run build    # ~5 сек, output → frontend/dist/
cp -r frontend/dist/* server/public/
systemctl restart indparkdocs   # Express раздаёт статику
```

Или через Nginx напрямую:
```nginx
location / {
    root /root/workspace-indparkdocs/frontend/dist;
    try_files $uri $uri/ /index.html;
}
location /api {
    proxy_pass http://127.0.0.1:3002;
}
```

## 9. Когда начинать

**Не сейчас.** Текущий приоритет — функциональность (фичи для Валентины).

Хороший момент для старта:
- Когда текущий backlog станет пустым (или стабильным)
- Когда появится фича, которую невозможно/мучительно делать на vanilla JS
- Когда фронтенд перевалит за ~25 000 строк

**Первый кандидат на миграцию — Заметки** (Excalidraw даст 10x улучшение).

## 10. Альтернатива: частичный React

Можно НЕ мигрировать весь фронтенд, а встроить React только для сложных компонентов:

```html
<!-- Старый HTML -->
<div id="notes-root"></div>
<script src="/app/notes-bundle.js"></script>
```

Vite собирает только `NotesPage` → `notes-bundle.js` (~200KB с Excalidraw). Остальной фронтенд остаётся vanilla.

**Плюсы**: минимум работы, максимум выгоды для конкретной фичи.
**Минусы**: два стека живут параллельно навсегда.
