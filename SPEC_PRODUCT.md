# IndParkDocs — Продуктовая спецификация

**Система:** IndParkDocs  
**Назначение:** Управление объектами, договорами и контрагентами индустриального парка  
**URL:** https://docs.zvezda-park.com  
**Версия спеки:** 1.0 (март 2026)

---

## 1. Зачем это нужно

IndParkDocs — внутренняя система АО «Индустриальный Парк Звезда» для:

- Учёта объектов недвижимости (корпуса, помещения, земельные участки)
- Ведения договоров аренды и связанных с ними документов
- Хранения и отслеживания оборудования и его принадлежности
- Аналитики: долги арендаторов, финансовые показатели из 1С, план/факт аренды
- AI-ассистента для получения данных на естественном языке

Система заменяет разрозненные Excel-таблицы единым источником правды.

---

## 2. Модули

### 2.1 Реестр сущностей (главная)
Центральный реестр всех объектов системы. Представление: список или карточки.  
Поиск по имени, номеру, контрагенту. Фильтрация по типу сущности.

### 2.2 Карта территории
Интерактивная SVG-карта территории парка.  
Корпуса и ЗУ отображаются как цветные полигоны. Клик → карточка объекта.  
Режим редактирования: рисование полигонов, подписи (short_name).

### 2.3 Поэтажные планы
Загрузка PDF-плана этажа. Рисование полигонов помещений поверх плана.  
Помещения окрашиваются по статусу аренды: свободно / занято / частично.

### 2.4 Финансы (1С)
Данные о движении денежных средств из 1С через OData API.  
Разделы: **Должники** (просроченная дебиторка), **Бюджет** (план/факт по ЦФО ИП), **Расходы** (по статьям).  
Данные кэшируются на 5 минут. Требует VPN-подключения к 1С.

### 2.5 Аналитика (OLAP-куб)
Произвольный срез данных по договорам/сущностям.  
Измерения: тип договора, контрагент, наше юрлицо, статус, год.  
Pivot-таблица с drill-down в отдельные записи.

### 2.6 Отчёты
- **Аренда** — сводная таблица договоров аренды с площадями и ставками
- **Анализ аренды** — интерактивный анализ с временной шкалой и drill-down
- **Связанные объекты** — сводка по связям (оборудование, договоры, помещения)
- **Должники** — список арендаторов с просроченной задолженностью из 1С

### 2.7 Зачёты (Legal)
Учёт актов зачёта взаимных требований между компаниями группы.  
Связь зачёта с конкретными договорами.

### 2.8 Письма и обращения
Реестр писем, сгруппированных по темам (топикам).  
Привязка письма к договору, контрагенту, объекту.

### 2.9 AI Аналитик
Чат-ассистент внутри системы на базе Claude Sonnet.  
Понимает вопросы на русском языке, выполняет SQL-запросы к БД (только SELECT),  
возвращает ответы с данными и объяснениями. Лимит: 300K токенов на сессию.

### 2.10 Настройки
Управление справочниками (значения dropdown-полей для каждого типа сущности).  
Управление пользователями. Настройка типов сущностей и их полей.

### Компании: свои и чужие

Поле `is_own`:
- `true` — собственная компания группы (ИПЗ, ЭКЗ, КГ Гермес и др.)
- `false` — контрагент (арендатор, поставщик, подрядчик)

Все 428 компаний в одной таблице. Наши юрлица используются как «наша сторона» в договорах.

## 4. Поля сущностей

### Договор
| Поле | Тип | Смысл |
|------|-----|-------|
| Тип договора | select | Аренды / Субаренды / Услуг / Подряда / Поставки / Эксплуатации / Купли-продажи / ТО и ППР / Процентного займа / Цессии / Аренда оборудования / Работы/Подряда |
| Номер | text | Номер договора |
| Наша сторона | select | Одна из собственных компаний группы |
| Контрагент | select | Арендатор / поставщик / подрядчик |
| Статус | select | Создан / Подписан / Архив |
| ВГО | авто | Внутригрупповая операция (обе стороны — наши компании) — вычисляется автоматически |
| Сумма / ставка | number | Зависит от типа договора |

**Поля объектов договора (5 типов):** помещения, ЗУ, части ЗУ, здания, оборудование — хранятся как DB-связи `located_in`.

### Доп. соглашение
Наследует от родительского договора: тип, стороны, юрлицо, контрагента.  
Собственные поля: номер ДС, дата, что изменилось, статус, ставки аренды.

### Оборудование
Категория (Электрооборудование / Газовое / Тепловое / Крановое хозяйство / Машины и механизмы / ИК / ЖД / Вентиляционное), инвентарный номер, серийный номер, год, производитель, статус (В работе / На ремонте / Законсервировано / Списано).  
Иерархия: оборудование может быть частью другого оборудования (агрегат → деталь).

### Счётчик
Тип (Электричество / Вода / Тепло / Газ), номер счётчика, место установки, дата установки, плановое потребление, ед. измерения, статус (Установлен / На поверке / Демонтирован).

## 5. Связи между сущностями

| Тип связи | Смысл | Пример |
|-----------|-------|--------|
| `party_to` | Сторона договора | Компания → Договор |
| `located_in` | Расположен в / объект договора | Помещение → Корпус; Договор → Помещение |
| `supplement_to` | Является ДС к | ДС → Договор |
| `subject_of` | Предмет договора (оборудование) | Оборудование → Договор |
| `on_balance` | На балансе компании | Оборудование → Компания |
| `installed_on` | Установлен на | Счётчик → Помещение / ЗУ |

Связи хранятся в таблице `relations`. При сохранении договора связи `party_to` и `located_in` пересоздаются автоматически.

## 6. Жизненный цикл договора

```
Договор (Создан)
  │
  ├── Подписан
  │     ├── Действующий
  │     │     ├── ДС №1 (изменение ставки)
  │     │     ├── ДС №2 (изменение помещений)
  │     │     └── Акт выполненных работ
  │     └── Архив (договор завершён)
  └── Архив (не исполнен)
```

**ВГО (внутригрупповая операция):** автоматически помечается, если обе стороны договора — собственные компании группы. Используется для исключения из внешних отчётов.

## 9. Интеграции

### 1С OData API
- **Адрес:** `http://192.168.2.3/BF/odata/standard.odata` (через VPN L2TP/IPsec)
- **Доступ:** только чтение (GET-запросы)
- **Данные:** ДДС (поступления/списания), дебиторская задолженность, бюджет
- **Кэш:** 5 минут на сервере
- **VPN watchdog:** `/usr/local/bin/vpn-watchdog.sh`, каждые 5 минут
- **При недоступности:** страница финансов показывает последние кэшированные данные

### Anthropic Claude API
- **Модель:** claude-sonnet-4-5 (AI Аналитик внутри системы)
- **Режим:** tool use (SQL-инструмент для SELECT-запросов к БД)
- **Лимит:** 300K токенов на сессию чата, timeout 90 секунд на вызов API
- **Доступ:** только авторизованные пользователи

## 10. Роли и права

| Роль | Возможности |
|------|------------|
| `admin` | Полный доступ: CRUD сущностей, управление пользователями, настройки |
| `editor` | Создание и редактирование сущностей, просмотр всего |

Аутентификация: JWT (access token 15 мин + refresh token 7 дней в httpOnly cookie).  
TOTP (2FA): опционально, настраивается на профиле пользователя.

## 11. Что отличает систему

1. **Единая модель сущностей** — все объекты в одной таблице `entities` с типом. Новый тип = строка в `entity_types`, без изменения схемы.
2. **Гибкие поля** — поля определяются в `field_definitions` и редактируются через UI без деплоя.
3. **Связи как данные** — все отношения между объектами хранятся в `relations` с типом. Легко добавить новый тип связи.
4. **Интеграция 1С** — живые финансовые данные, импорт компаний через ИНН/КПП.
5. **AI-ассистент** — SQL-агент с доступом к реальным данным, отвечает на вопросы бизнеса.
