# БИТ.Финанс — Доступ через 1С OData

## Подключение

- **Endpoint**: `http://192.168.2.3/BF/odata/standard.odata/`
- **Credentials**: `АртамоноваВП` / `впа` (HTTP Basic Auth)
- **⚠️ Пользователь `odata.user` НЕ имеет доступа к бюджетным данным** — только `АртамоноваВП`
- **Формат**: добавлять `$format=json` ко всем запросам
- **VPN**: Запросы работают только с сервера (VPN-туннель до Звезда Парка)

### Поднятие VPN-туннеля (если ppp0 не активен)
```bash
echo "c zvezda" > /var/run/xl2tpd/l2tp-control
sleep 8
ip route add 192.168.2.0/24 via 10.253.190.1 dev ppp0 2>/dev/null
# Проверка: ip addr show ppp0
```

## Статьи оборотов (строки бюджета)

**Endpoint**: `Catalog_бит_СтатьиОборотов`

Иерархический справочник. Папки верхнего уровня:
- **БДДС** → ПОСТУПЛЕНИЯ, РАСХОДЫ, ПРИТОК (ОТТОК) ДС, САЛЬДО
- **БДР** → ДОХОДЫ, РАСХОДЫ, НАЛОГИ ИЗ ПРИБЫЛИ, ФИНАНСОВЫЙ РЕЗУЛЬТАТ
- Каждая папка содержит конкретные статьи (аренда, зарплата, и т.д.)

Запрос статей:
```
Catalog_бит_СтатьиОборотов?$format=json&$select=Ref_Key,Description,Parent_Key,IsFolder
```

## Основные регистры данных

### 1. Обороты по бюджетам (ПЛАН и ФАКТ)
**Endpoint**: `AccumulationRegister_бит_ОборотыПоБюджетам`

⚠️ **ОГРОМНЫЙ регистр!** Не запрашивать без фильтров — зависнет.

Поля (измерения):
- `Period` — дата
- `Сценарий_Key` — ссылка на сценарий (ПЛАН/ФАКТ)
- `ЦФО_Key` — ссылка на ЦФО (Catalog_Подразделения)
- `СтатьяОборотов_Key` — ссылка на статью бюджета
- `Организация_Key` — ссылка на организацию
- `Контрагент_Key`, `ДоговорКонтрагента_Key`, `Проект_Key` — доп. аналитики
- `БанковскийСчет_Key` — банковский счёт

Ресурсы (суммы):
- `Сумма` — сумма в валюте
- `СуммаРегл` — сумма регламентированная
- `СуммаУпр` — сумма управленческая
- `Количество` — количество

### 2. Заявки на расходование средств (Казначейство)
**Endpoint**: `Document_бит_ЗаявкаНаРасходованиеСредств`

5000+ документов, все компании. Работает с `$top`.

Поля:
- `Date`, `Number`, `Posted`
- `Организация_Key` — организация
- `ЦФО_Key` — ЦФО
- `СуммаДокумента` — сумма
- `СтатьяОборотов_Key` — статья бюджета
- `Контрагент_Key` — контрагент
- `БанковскийСчет_Key` — банковский счёт

### 3. Формы ввода бюджета
**Endpoint**: `Document_бит_ФормаВводаБюджета`

Документы ввода бюджетных данных. Содержат табличную часть БДДС.

### 4. Бюджетные операции
**Endpoint**: `Document_бит_БюджетнаяОперация`

### 5. Планируемые поступления/расходы ДС
- `AccumulationRegister_бит_ПланируемоеПоступлениеДенежныхСредств`
- `AccumulationRegister_бит_ПланируемыйРасходДенежныхСредств`

### 6. Сведения об остатках ДС

## Важные ограничения (OData quirks)

1. **`$select` ломает некоторые запросы** — возвращает 0 записей. Если `$select` не работает — убрать его
2. **`$top` без `$orderby`** может возвращать 0 записей для некоторых документов. Добавлять `$orderby=Date desc`
3. **Листинг Catalog_Подразделения запрещён** — но прямой GET по GUID работает
4. **`odata.user` не видит бюджетные данные** — только `АртамоноваВП`
5. **Большие регистры зависают** — всегда использовать `$top` и/или `$filter`
6. **Кодировка**: URL-encode кириллицу в `$filter`

## Примеры запросов

### Получить сценарии планирования
```
GET Catalog_СценарииПланирования?$format=json&$select=Ref_Key,Description
```

### Получить справочник бюджетов  
```
GET Catalog_бит_Бюджеты?$format=json&$select=Ref_Key,Description,DeletionMark
```

### Получить статьи оборотов (иерархия)
```
GET Catalog_бит_СтатьиОборотов?$format=json&$select=Ref_Key,Description,Parent_Key,IsFolder
```

### Получить заявки на расходование за 2026
```
GET Document_бит_ЗаявкаНаРасходованиеСредств?$format=json&$top=5000&$orderby=Date desc
```

### Получить конкретный ЦФО по GUID
```
GET Catalog_Подразделения(guid'7082ad6f-5774-11e9-9063-d094662a4ed7')?$format=json
```

## Стратегия получения бюджетных данных

Рекомендуемый подход для дашборда План/Факт:

1. **Загрузить справочники** (кэшировать):
   - Сценарии (`Catalog_СценарииПланирования`)
   - Бюджеты (`Catalog_бит_Бюджеты`)
   - Статьи оборотов (`Catalog_бит_СтатьиОборотов`)
   - ЦФО — использовать маппинг из этого файла

2. **Загрузить данные** из `AccumulationRegister_бит_ОборотыПоБюджетам`:
   - Фильтровать по `Сценарий_Key` (ПЛАН 2026 / ФАКТ)
   - Использовать `$top` + пагинацию
   - Группировать по `СтатьяОборотов_Key` + `ЦФО_Key` + `Period`

3. **Альтернативный подход** (если регистр слишком тяжёлый):
   - Использовать `Document_бит_ФормаВводаБюджета` для плановых данных
   - Использовать `Document_бит_ПолучениеФактическихДанных` для факта

4. **Для казначейства**:
   - `Document_бит_ЗаявкаНаРасходованиеСредств` — заявки (работает стабильно)
   - `Document_бит_СведенияОбОстаткахДенежныхСредств` — остатки
