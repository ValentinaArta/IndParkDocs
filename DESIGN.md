# DESIGN.md — Правила дизайна портала IndParkDocs

> Этот файл — единственный источник правил UI/UX. При разработке сверяться здесь.
> Обновлять при каждом дизайн-решении, чтобы не объяснять одно и то же дважды.

---

## 1. Иконки

**Единственный способ рисовать иконки:**
```js
icon('название-из-lucide')          // стандартный размер
icon('название-из-lucide', 16)      // явный размер (px)
entityIcon('contract')              // иконка типа сущности
```

**Библиотека:** [Lucide](https://lucide.dev/icons/) — SVG, не зависит от ОС.

**Маппинг частых иконок:**
| Смысл | Lucide name |
|---|---|
| Предупреждение / ошибка | `alert-triangle` |
| Удалить / закрыть | `x` |
| Подтверждение / ОК | `check` |
| Успех (зелёный) | `check-circle-2` |
| Ошибка (красный) | `x-circle` |
| Настройки | `settings` |
| Документ | `file-text` |
| Буфер обмена | `clipboard` |
| Почта | `mail` |
| Телефон | `phone` |
| Ссылка / связь | `link` |
| Поиск | `search` |
| Сохранить | `save` |
| Загрузка файла | `upload` |
| Деньги / сумма | `banknote` |
| Предупреждение (авария) | `alert-circle` |
| Редактировать | `pencil` |
| Добавить | `plus` |
| Меню / список | `list` |
| Дерево / иерархия | `git-branch` |
| Карта | `map` |
| Замок / безопасность | `lock` |
| Аналитика | `bar-chart-2` |

**❌ НЕЛЬЗЯ использовать:**
- Emoji (⚠️ ✅ ❌ 📋 ⚙️ и т.д.) — выглядят по-разному на iOS/macOS/Windows
- Символы Unicode как иконки (✕ ☰ ⚖ и т.д.)
- Внешние иконочные шрифты (FontAwesome, Material Icons)

**Цветовые маркеры-точки** (статусы, ВГО и т.д.):
```html
<!-- Вместо 🔴🟡🔵 — CSS-кружки -->
<span style="display:inline-block;width:10px;height:10px;border-radius:50%;background:#ef4444"></span>
<span style="display:inline-block;width:10px;height:10px;border-radius:50%;background:#f59e0b"></span>
<span style="display:inline-block;width:10px;height:10px;border-radius:50%;background:#3b82f6"></span>
```

## 2. Цвета

Используем CSS-переменные из темы. **Не хардкодить hex-цвета** без необходимости.

| Переменная | Назначение |
|---|---|
| `var(--accent)` | Основной акцент (синий) |
| `var(--bg-card)` | Фон карточки |
| `var(--bg-hover)` | Фон при hover / readonly |
| `var(--bg-secondary)` | Вторичный фон |
| `var(--border)` | Граница элементов |
| `var(--text-primary)` | Основной текст |
| `var(--text-secondary)` | Второстепенный текст |
| `var(--text-muted)` | Серый/подсказка |
| `var(--danger)` | Красный (ошибка, удаление) |
| `var(--success)` | Зелёный (успех) |
| `var(--warning)` | Жёлтый (предупреждение) |

Семантические хардкодные цвета (допустимо для статусов):
- Красный: `#ef4444` (авария, ошибка)
- Жёлтый: `#f59e0b` (предупреждение)
- Зелёный: `#22c55e` (норма, успех)
- Синий: `#3b82f6` (ВГО, информация)

## 3. Типографика

- **Основной шрифт:** системный стек (не подключать внешние)
- **Размеры:**
  - Заголовок секции: `11px`, `font-weight:600`, `text-transform:uppercase`, `letter-spacing:.5px`
  - Метка поля (`<label>`): `13–14px`
  - Основной текст: `14px`
  - Мелкий текст / подсказка: `12px`, `color:var(--text-muted)`
- **Жирный текст** (`font-weight:600`) — только для акцентов, не для обычного контента

## 4. Компоненты форм

### Поля ввода
```html
<input>     <!-- стандартный input, стили из CSS -->
<textarea>  <!-- резиновый: style="resize:vertical;min-height:60px" -->
<select>    <!-- стандартный select -->
```

### Кнопки
```html
<button class="btn">Отмена</button>
<button class="btn btn-primary">Сохранить</button>
<button class="btn btn-danger">Удалить</button>
```
- **Не создавать кастомные стили кнопок** без необходимости
- Размер кнопок единообразный — через класс, не inline-стилями

### Группа полей
```html
<div class="form-group">
  <label>Название поля</label>
  <!-- input/select/etc -->
</div>
```

### Секция формы (3-секционный layout договора)
```html
<div class="form-section">
  <div class="form-section-title">ЗАГОЛОВОК</div>
  <div class="form-section-body">
    <!-- поля -->
  </div>
</div>
```

## 5. Модальные окна

- Заголовок: `<h3>` внутри модала
- Кнопки внизу: `<div class="modal-actions">` с Отмена + Действие
- Ширина: стандартная (CSS), переопределять только через `setModalSize()`

---

## 6. Таблицы

- **Не использовать `<table>`** для списков сущностей — только `div`-based layout
- Таблицы допустимы для отчётов (данные с заголовками)
- Заголовки таблиц: `font-weight:600`, `font-size:12px`
- Ячейки: `padding:8px 12px`

---

## 7. Списки сущностей

- Каждая строка — `<div class="entity-row">` (или аналог)
- Иконка типа через `entityIcon(typeName)`
- Название — ссылка `onclick="showEntity(id)"`
- Бейдж статуса — `_docStatusBadge(status)` из `core/utils.js`

---

## 8. Правила кода

- **Экранирование:** всегда `escapeHtml()` для пользовательских данных в HTML
- **Числа:** `_fmtNum(n)` для форматирования с разделителями
- **Даты:** `_fmtDate(d)` → `DD.MM.YYYY`
- **API calls:** `api(path, options)` из `core/api.js`
- **Нет `document.write`**, нет `innerHTML` без `escapeHtml`

---

## 9. Адаптивность

- Мобильный минимум: **320px**
- Планшет: **768px**
- Десктоп: **1280px+**
- Горизонтальный скролл на мобильных — **недопустим**
- Интерактивные элементы: минимум **44×44px** для нажатия пальцем
